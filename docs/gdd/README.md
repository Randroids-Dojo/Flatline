# Flatline GDD (2026-07 reboot)

Canonical design document for Flatline: a black-and-white 1930s rubber-hose cartoon noir FPS with original Doom mechanics, an endless streamed dungeon, and Rogue Legacy 2 style death-driven meta progression.

The previous game (single-arena wave shooter) is archived under `docs/_archive/2026-07-07-pre-reboot/`. Its GDD no longer governs anything.

Each section file carries a `Status:` line and a `### Build log`. Sections:

- `01-vision.md`: what the game is.
- `02-design-pillars.md`: the four pillars.
- `03-platform-stack.md`: desktop web, Next.js + Three.js + Vercel.
- `04-core-loop.md`: run, die, spend, repeat.
- `05-endless-dungeon.md`: chunked on-the-fly generation.
- `06-movement.md`: Doom momentum model.
- `07-weapons.md`: the seven-slot arsenal.
- `08-combat.md`: hitscan, projectiles, splash, armor.
- `09-enemies.md`: the bestiary and AI.
- `10-doors-vaults.md`: Doom doors, locked vaults, keys.
- `11-pickups.md`: health, armor, ammo, cheddar.
- `12-meta-progression.md`: the Case Board, Armory, and rent.
- `13-relics.md`: one-run contraband.
- `14-art-direction.md`: rubber-hose ink style, procedural drawing.
- `15-film-post.md`: grain, diffusion, presets.
- `16-hud-mugshot.md`: status bar and reactive face.
- `17-automap.md`: hold-Tab map.
- `18-audio.md`: synthesized sfx and ambience.
- `19-screen-flow.md`: title, run, death, office.
- `20-persistence.md`: save schema.
- `21-testing.md`: unit and e2e strategy.
- `22-performance.md`: budgets.
- `99-out-of-scope.md`: cut lines.
